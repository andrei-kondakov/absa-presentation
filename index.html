Актуальность проблемы

  • Задача анализа тональности является одной из активно развиваемых технологий в сфере автоматической обработки текстов.
  • Развитие социальных сетей, онлайновых рекомендательных сервисов.
  • Использование алгоритмов машинного обучения для анализа тональности текстов в настоящее время позволяет достигнуть лучших результатов.
Задача анализа тональности, т.е. выявления мнений автора текста по поводу предмета, обсуждаемого в тексте, является одной из активно развиваемых технологий в сфере автоматической обработки текстов в последние годы. Актуальность проблемы связана с ростом социальных сетей, онлайн сервисов с отзывами в сферах услуг, которые содержат большое количество мнений по разным вопросам.

Цель работы

Исследовать алгоритмы из области компьютерной лингвистики и машинного обучения для решения задачи автоматического извлечения тональности текстов.

С исследовательской точки зрения работа предоставляет большой интерес, т.к. предполагает решения комплекса нетривиальных задач из области компьютерной лингвистики и машинного обучения.

Постановка задачи

  • Применение алгоритмов машинного обучения, включая нейронные сети, для извлечения аспектов и тональности предложений в отзывах о ресторанах.
  • В качестве размеченных выборок для обучения и тестирования использовать данные с соревнования Semeval 2016.
  • Сравнение результатов с существующими подходами.
Постановка задачи анализа тональности разнообразна: определение тональности всего текста, отдельных предложений, конкретных фраз и терминов. Более формально проблема анализа тональности называется аспектно-эмоциональным анализом, где аспект – это интересующая сущность и его признак или категория.

Основные подзадачи

  1. Определение аспектов в предложении.
  2. Определение тональности аспекта.

Прим. Аспект – это интересующая сущность.

Исходные данные

  • Размеченные выборки с отзывами на русском языке о ресторанах.
  • Предоставлены организаторами на международном семинаре по семантической оценки SemEval-2016.
  • Обучающая выборка: 300 отзывов (~4000 предложений).
  • Тестовая выборка: 100 отзывов (~1000 предложений).
  • Формат: XML.

Эксперимент №1: алгоритм обучения классификаторов

  • Минимальная предобработка текстов.
  • Бинарный векторайзер предложений (размер вектора - 1000).
  • Для каждого аспекта один и тот же алгоритм обучения
  • Оценка: макроусреднение F1-мер классификаторов.

Результаты

Вывод: Метод опорных векторов показал наилучший результат из тестируемых.
Поэтому он будет использован в качестве алгоритма для обучения классификаторов аспектов.

RF (random forest) — это множество решающих деревьев. В задаче регрессии их ответы усредняются, в задаче классификации принимается решение голосованием по большинству. Все деревья строятся независимо по следующей схеме: Выбирается подвыборка обучающей выборки размера samplesize (м.б. с возвращением) – по ней строится дерево (для каждого дерева — своя подвыборка). Для построения каждого расщепления в дереве просматриваем max_features случайных признаков (для каждого нового расщепления — свои случайные признаки). Выбираем наилучшие признак и расщепление по нему (по заранее заданному критерию). Дерево строится, как правило, до исчерпания выборки (пока в листьях не останутся представители только одного класса), но в современных реализациях есть параметры, которые ограничивают высоту дерева, число объектов в листьях и число объектов в подвыборке, при котором проводится расщепление.

Эксперимент №2: удаление стоп-слов
